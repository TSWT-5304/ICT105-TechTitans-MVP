# MVP Experiment Plan

## 1. Group and Project Information
- **Group name:** TechTitans
- **Project title:** PeerTutor – Student-to-Student Tutoring Marketplace
- **Repository link:** https://github.com/TSWT-5304/ICT105-TechTitans-MVP
- **Main target user:** Rangsit University students seeking academic tutoring support, student tutors, and platform administrators.
- **Prototype platform:** Frontend Web Application (HTML, CSS, JavaScript) with Google Sheets for prototype data storage.

## 2. Experiment Objective
> We want to evaluate whether students, tutors, and administrators can successfully complete the core PeerTutor workflows—including tutor discovery, tutoring requests, tutor registration, payment simulation, and administrator management—without guidance using the MVP prototype.

## 4. MVP Experiment Type
### Selected Experiment Type
- Clickable Prototype
- Simple Web Prototype
- Form-based Simulation
- Dashboard Demo

### Reason for Selection
The PeerTutor MVP is implemented as an interactive frontend web application using HTML, CSS, and JavaScript. The experiment will evaluate whether users can successfully complete the primary Student, Tutor, and Administrator workflows while interacting with realistic interface components and simulated data stored through Google Sheets. This approach allows the team to validate usability, navigation, and workflow effectiveness before implementing a full backend system.

## 5. Test Users

**Total Testers:** **10**



## 6. Experiment Procedure Summary
The experiment will be conducted using the interactive PeerTutor prototype.
Each participant will complete tasks based on their assigned role:

### Student Tasks
- Access the homepage.
- Register or log in.
- Search and filter tutors.
- View a tutor profile.
- Submit a tutoring request.
- Complete the payment simulation.
- View payment history.
- Leave a review after the tutoring session.

### Tutor Tasks
- Register as a tutor.
- Submit a tutor application.
- Manage tutor profile.
- Review incoming tutoring requests.
- Approve or reject requests.
- View payment status and earnings.

### Administrator Tasks
- Log in to the Admin Dashboard.
- Review tutor applications.
- Approve or reject tutor applications.
- Verify student payments.
- Manage tutor earnings.
- Review dashboard analytics.

During the experiment, the team will observe task completion, navigation flow, user feedback, completion time, and any usability issues encountered.

## 7. Expected Learning
After completing the MVP experiment, the team expects to determine:
- Whether students can successfully find suitable tutors and complete the tutoring request process.
- Whether tutors can complete registration and manage tutoring requests effectively.
- Whether administrators can efficiently manage users, tutor applications, payments, and dashboard information.
- Whether the navigation, page layout, and workflow are intuitive without additional guidance.
- Which usability issues should be improved before the final prototype demonstration.
- Which features require refinement before future implementation beyond the MVP.

The findings from this experiment will be used to improve the user experience, validate the MVP features, and guide the implementation of the final PeerTutor prototype.
