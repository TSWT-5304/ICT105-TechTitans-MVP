## Wireframe Quality Rules

- Maintain a consistent navigation bar, color scheme, typography, icons, buttons, and layout across every screen.
- Every wireframe must support a meaningful user task and directly correspond to the User Stories, MVP Feature List, and Functional Requirements defined in Lab 04.
- Use realistic sample data, including tutor names, subjects, hourly rates, ratings, tutoring request statuses, payment records, and dashboard statistics.
- Display appropriate validation messages, confirmation dialogs, and error prevention for user inputs.
- Ensure responsive layouts for desktop, and mobile devices.
- Include clear navigation between related screens to support the complete Student, Tutor, and Administrator workflows.
- Authentication screens (Role Selection, Login, and Sign Up) support secure access to the platform but are not considered separate MVP features.
