# Experiment Script
##  1. Opening Script
Thank you for participating in our usability test for **PeerTutor – Student-to-Student Tutoring Marketplace**.

We are testing our prototype, **not your abilities**. There are no right or wrong answers. We would like you to use the prototype as naturally as possible and **think aloud** while completing each task. If you become confused, please explain what you are thinking so we can better understand your experience.

Your feedback will help us improve the usability and functionality of the PeerTutor platform before the final implementation.

## 2. Test Scenario
Imagine you are a **Rangsit University student** who needs academic tutoring for an upcoming examination. You decide to use **PeerTutor** to search for a suitable tutor, compare tutor profiles, submit a tutoring request, complete the payment process, and monitor your tutoring request. If you are assigned the Tutor or Administrator role, please complete the tasks based on that role.

# 4. Observation Notes
During the usability test, the facilitator should record observations including:
- Whether the tester completes each task successfully.
- Areas where the tester hesitates or becomes confused.
- Incorrect clicks or navigation mistakes.
- Questions asked by the tester.
- Time required to complete each task.
- User comments regarding layout, wording, navigation, or interface design.
- Suggestions for improving the overall user experience.

Special attention should be given to:
- Tutor Search and Filtering
- Tutoring Request Form
- Online Payment Workflow
- Student Dashboard
- Tutor Dashboard
- Administrator Dashboard
- Overall navigation between screens

# 5. Closing Questions
After completing the tasks, ask the tester the following questions:
1. What was the easiest part of using PeerTutor?
2. Which part of the prototype was confusing or difficult to use?
3. Was it easy to find a suitable tutor?
4. Did the tutor profile provide enough information before making a tutoring request?
5. Was the tutoring request process straightforward?
6. Was the payment process easy to understand?
7. Were the request status labels clear?
8. Which feature did you find most useful?
9. Which feature needs the most improvement?
10. Would you consider using PeerTutor instead of Facebook or LINE groups to find a tutor? Why or why not?
11. Do you have any additional suggestions that would improve the platform before the final prototype?

# Facilitator Reminder
- Do not guide the tester unless they become completely stuck.
- Encourage the tester to think aloud throughout the session.
- Record both positive and negative feedback objectively.
- Thank the participant after completing the experiment.
