# 1. Validation Objective
The objective of this customer validation was to evaluate whether the PeerTutor MVP enables students, student tutors, and administrators to successfully complete the platform's core workflows with minimal guidance.

The validation focused on usability, navigation, workflow completion, task efficiency, and user satisfaction while confirming that the implemented features satisfy the Functional Requirements (FR-01 to FR-19). The findings will help determine whether the current MVP is suitable for further development and identify areas requiring improvement before the final prototype demonstration.

# 4. Validation Procedure
Each participant was assigned tasks according to their role.

### Student Tasks
- Open the homepage.
- Select the Student role and log in.
- Search and filter tutors.
- View tutor profiles.
- Submit a tutoring request.
- Complete the payment simulation.
- View payment history.
- Leave a review.
### Student Tutor Tasks
- Register as a tutor.
- Submit a tutor application.
- Create and manage a tutor profile.
- Review tutoring requests.
- Approve or reject tutoring requests.
- View payment status and tutor earnings.

### Administrator Tasks
- Review tutor applications.
- Approve or reject tutor applications.
- Manage user accounts.
- Verify student payments.
- Review tutor earnings.
- View dashboard analytics.

During testing, the facilitator observed task completion, navigation behaviour, completion time, user comments, and any usability issues.
