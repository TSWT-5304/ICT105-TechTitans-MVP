# Wireframe Usability Review Checklist
Use this checklist before submitting Lab 05.
| Check Item | Yes/No | Evidence or Screenshot |
|---|---|---|
| The homepage clearly states the problem and main action. | Yes | homepage.png |
| The main user pathway can be followed from start to finish. | Yes | Homepage → Tutor Search → Tutor Profile → Request Form → Status → Review |
| The input form includes required fields and validation feedback. | Yes | request-form.png, tutor-registration.png |
| The record list includes search/filter/category controls. | Yes | tutor-search.png |
| The detail view shows full information and current status. | Yes | tutor-profile.png, student-dashboard.png |
| The admin view includes at least one management action. | Yes | admin-approval.png (Approve/Reject Tutor, Suspend User) |
| The dashboard shows meaningful summary information. | Yes | admin-dashboard.png (Pie Chart & Line Graph), student-dashboard.png, tutor-dashboard.png |
| Navigation is consistent across screens. | Yes | Common navigation bar, buttons, and layout across all wireframes |
| The design uses realistic sample data. | Yes | Sample tutor profiles, tutoring requests, ratings, request statuses, dashboard statistics |
| Each screen maps to system-requirements.md. | Yes | Verified through Feature-Requirement Mapping (FR-01 to FR-16) |
