# Feature-Value Mapping

| Feature ID | MVP Feature | Requirement ID | User Story ID | User Value | Business / Operational Value | Prototype Screen or Module | Priority |
|---|---|---|---|---|---|---|---|
| F-01 | Homepage / Landing | FR-01 | US-01 | Students immediately understand what PeerTutor does and how to start | Sets clear entry point, reduces bounce/confusion for first-time visitors | index.html | Must |
| F-02 | Tutor Search & Filter | FR-02, FR-06 | US-02 | Students find a matching tutor in seconds instead of asking around | Demonstrates core marketplace value; drives request volume | tutors.html | Must |
| F-03 | Tutor Profile (View) | FR-07 | US-03 | Students can judge tutor fit (experience, rating, price) before committing | Builds trust, reduces mismatched/failed bookings | tutor-detail.html | Must |
| F-04 | Tutoring Request Form | FR-03, FR-10, FR-11 | US-04 | Students submit a structured request instead of an informal chat message | Produces clean, structured demand data admin/tutors can act on | request.html | Must |
| F-05 | Confirmation & Status Page / Student Dashboard | FR-08, FR-11, FR-12 | US-05 | Students know their request was received and can track progress | Reduces repeat inquiries; shows request pipeline health | student-dashboard.html | Must |
| F-06 | Review & Rating Page | FR-02 | US-06 | Future students get honest signals about tutor quality | Builds reputation system that increases repeat platform use | review.html | Should |
| F-07 | Tutor Registration / Application Form | FR-03, FR-10, FR-11 | US-07 | Tutors get a clear, one-time path to offer their services | Grows tutor supply in a structured, reviewable way | tutor-register.html | Must |
| F-08 | Tutor Dashboard (application status + incoming requests) | FR-08, FR-05, FR-12 | US-08, US-10 | Tutors know if they're approved and can manage requests in one place | Keeps tutors engaged and responsive, improving fulfillment rate | tutor-dashboard.html | Must |
| F-09 | Tutor Profile Management | FR-05, FR-07 | US-09 | Tutors control how they're presented to students | Improves match quality and platform credibility | tutor-profile-edit.html | Must |
| F-10 | Admin User & Tutor Approval Management | FR-05, FR-07, FR-09 | US-11, US-12 | Students trust that listed tutors are vetted | Protects platform quality/safety; core moderation control point | admin.html | Must |
| F-11 | Admin Dashboard (Analytics) | FR-12 | US-13 | (Indirect) students benefit from a well-monitored, improving platform | Gives admin/university visibility into demand, adoption, and bottlenecks | admin-dashboard.html | Should |
