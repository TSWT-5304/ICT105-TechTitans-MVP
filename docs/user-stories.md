# Lab 04 - User Stories and Acceptance Criteria
## User Story Format
As a [user role], I want to [goal/action], so that [benefit/value].
## User Stories
| Story ID | Role | User Story | Related Requirement | Priority | Acceptance Criteria | Demo Evidence |
|---|---|---|---|---|---|---|
| US-01 | Student | As a student, I want to view the homepage, so that I can understand the purpose of PeerTutor and access the system. | FR-01 | Must | Given I open the PeerTutor website, when the homepage loads, then I can see the platform name, navigation menu, introductory information, and the "Find a Tutor" and "Get Started Now" buttons.  | Screenshot of homepage |
| US-02 | Student | As a student, I want to search for tutors by subject, so that I can quickly find someone who can help with my specific course instead of browsing everywhere. | FR-05, 06 and 07 | Must | Given I am on the tutor listing page, when I type a subject name or select a category, then the list updates to show only tutors who cover that subject within 2 seconds. | Tutor search demo |
